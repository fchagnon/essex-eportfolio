# LIC: Launch Into Computing

MSc Cyber Security, University of Essex Online. This module provides a
broad foundation across computing, from logic and algorithms through
software engineering, data science, AI, cybersecurity, and enterprise IT
governance, culminating in a two-part End of Module Assignment
(Programming Exercise and Reflective Presentation).

Each unit folder contains its own README covering the unit's topic,
learning outcomes, formative assignment, and reflection, along with any
supporting code, data, and submitted work. Units 1 and 4 anchor a
four-week summative collaborative discussion that ran across the first
third of the module; Units 7, 9, and 10 were formative assignments
deliberately not submitted, each for a distinct, documented reason.

## Highlights

**[Unit 12: End of Module Assignment](Unit%2012)** — graded 82%
(Distinction). Part A empirically compares C and Python sorting
implementations across 800,000 records, demonstrating that algorithmic
complexity, not language choice, determines scalability. Part B argues
cybersecurity is transitioning from craft to engineering science, an
argument that went on to shape a published LinkedIn article, a keynote
address at Info-Tech LIVE, and ongoing applied research into a new
security operations operating model.

**[Unit 5: Data Science and Storage](Unit%205)** — a SQL versus NoSQL
analysis grounded in real historical finisher data from Paris-Brest-Paris,
the 1,200km randonneuring event the author intends to ride in 2027.

**[Unit 6: Principles of AI I](Unit%206)** — an NHL game-outcome
prediction model that caught its own baseline model "cheating" by
exploiting class imbalance, then used a corrected model to test, and
substantially explain, the mechanism behind home-ice advantage.

**[Unit 3: Principles of Computer Science](Unit%203)** — a Bubble Sort
versus Quick Sort comparison with real benchmark data, including an
empirical finding (reverse-sorted data outperforming random data)
explained through CPU branch prediction rather than algorithmic
complexity alone.
