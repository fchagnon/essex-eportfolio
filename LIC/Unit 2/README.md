# Unit 2: Logical Foundations of Computing: Boolean Algebra, Gates, and Set Theory

## Unit Topic

This unit covers Boolean logic, logic gates, and set theory as the
foundation of computational systems, programming languages, and
algorithm design, including their application in digital circuits,
database design, and enterprise computing.

## Learning Outcomes

On completion of this unit, I should be able to:

- Identify and critically analyse the role of Boolean logic and set
  theory in solving computing challenges.
- Gather and synthesise information on logic and mathematical
  applications in enterprise systems.
- Evaluate tools and techniques (e.g., logic gates, truth tables) to
  solve computational challenges.

## Reflection

**Identifying and critically analysing the role of Boolean logic.**
Responding to a classmate's post on AI in healthcare, I applied this
unit's antecedent/consequent framing directly to a real clinical AI
scenario: an AI scribe processing patient dialogue is, in formal logic
terms, handling the antecedents, the data and notes, while the
consequent, the actual clinical diagnosis, still requires human intuition
to account for variables a binary or probabilistic model might miss
(Mosley and Baltazar, 2019). That distinction gave me a genuinely useful
way to argue for a specific position (AI should remain a high-fidelity
assistant rather than a primary decision-maker) using this unit's own
logical structure, rather than a general appeal to caution.

**Gathering and synthesising information on logic in enterprise
systems.** This reply drew on a real, secondhand example rather than
only the assigned literature, a colleague's use of an AI scribe device in
general practice, including a "conditioning effect" where the AI's
habit of treating every data point as potentially significant began
influencing the practitioner's own manual note-taking. That's a genuine
data point about how formal-logic-style thinking can propagate from a
tool into human practice, not something the reading itself covered.

**Evaluating tools and techniques.** This reply engaged with the
*concept* of formal logical structure (antecedent and consequent)
applied to a real decision, rather than working through specific tools
like truth tables or logic gates directly. That's a genuine gap worth
naming honestly, the practical, notation-level side of this unit's
content is not represented in this particular piece of work.
